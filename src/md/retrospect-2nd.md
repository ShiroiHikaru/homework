### 웹접근성과 웹표준

본격적으로 html 수업을 시작하면서 먼저 '웹(Web)'이란 무엇인지, 또 html과 xhtml의 차이점 등 이론 수업을 진행하였다.

한편으로는 들어본 적 있는 **웹접근성과 웹표준**에 대해 짚어보는 수업이였다.
사실 들어본적도 있고 지켜야 한다는건 익히 알고 있었으나 정작 실무에서나 내 개인프로젝트에서나 **"작동하기만 하면 된다"**에 치우쳐 있어서 미처 챙기지 못하는 부분이였는데 중요성을 강조하는 내용과 더불어 **"WAI-ARIA"**도 함께 알아가는 기회였다.

---

```
WAI-ARIA란?
접근가능한 리치 인터넷 어플리케이션, 웹콘텐츠와 웹 애플리케이션의 접근성을 향상시키기 위한 W3C 기술명세서.

```

HTML과 XML의 차이점으로는
HTML의 경우 원래라면 문법 오류의 문제지만 유동성 있게 넘어가는 편에 반헤
XML의 경우 규칙을 매우 엄격하게 준수해야하고 소문자만을 지원한다.

또 API의 의미를 몰랐었는데 Application Programming interface 라는 뜻을 알았고
이것은 "자바스크립트로 미리 만들어둔 기능"이라는 의미에 대해서도 알게되었다.

다시 한번 정말 의미도 모르고 그저 사용 만 해왔구나..라는 심정을 느꼈다.
