Form 태그는 말 그대로 입력 폼(아이디, 비밀번호, 이메일 등)을 작성하는 태그 구성이기 때문에 웹사이트에서는 매우 중요한 태그라고 생각한다.

비단 로그인 페이지 뿐만 아니라 회원가입, 문의게시판에도 사용할 수 있기 때문에.

가장 기억에 남는 부분은 <fieldset> 태그를 있는 그대로 생각해서 비유했는데 칭찬을 받은거 같아 매우 뿌듯했다. 내가 비유한거지만 내가 놀랐다.

```
같은 빈 상자의 의미일지라도 <div>태그의 경우 아무 그림이 없는 박스라고 생각하면 쉽다. 아무 그림이 없는 박스에는 무엇을 담아도 그냥 박스니까.

반면 <fieldset> 태그의 경우 **연관성이 있는 폼 서식**만을 담을 수 있다.
<fieldset>은 비유를 하면 청송사과라고 적혀 있는 박스라 사과 만 담는 것이 베스트이다.
```

~사실 박스에 그림이 그려져 있어도 물건은 담을 수 있지만~

### Form 태그 type

|  타입명  |          의미          |
| :------: | :--------------------: | -------------------------------------------- |
|   text   | 텍스트(기본 디폴트 값) |
|  email   |         이메일         |
|   tel    |        전화번호        | = 모바일 특화 태그, 클릭 시 전화 키패드 노출 |
|   url    |        url 주소        |
|  radio   |      라디오 버튼       | = 1개만 선택 가능                            |
| checkbox |        체크박스        | = 복수선택 가능                              |
|  search  |          검색          |
|   file   |          파일          | = 파일첨부                                   |
|  reset   |          리셋          | = 클릭 시 입력 정보 초기화                   |
|  submit  |          전송          |
|  button  |          버튼          |

```
<form action="post" method="">
  <div>
    <label for="userName">유저이름</label>
    <input type="text" id="userName" name="userName" value="name">
  </div>
</form>

```

- 여러가지 항목에서 원하는 값 선택 => radio / check

- 목록 형태로 펼쳐지는 항목 => combo box

- 성격은 여러가지, 요소는 <input> 하나만 남는다. 단 input type="" 속성값에 따라 역할이 달라진다.

- input에는 반드시 1:1로 대응되는 lable을 가지고 가야한다.

- placeholder는 레이블이 아니다. 레이블의 역할은 lable 태그

- fieldset은 연관성이 있는 항목을 묶을 때 사용한다.

- label 태그 안에 input 태그를 추가 할 경우 screen reader에서 해당 서식이 연결되어 있음을 암묵적으로 알려준다.

- UserName : 타이틀 케이스, 파스칼 케이스 | userName : 카멜케이스 | user_name : 스네이크 케이스 | user-name : 케밥케이스

- 포커스를 지정할 수 없는 태그에도 tabindex 속성을 부여하면 활성화 할 수 있다.
