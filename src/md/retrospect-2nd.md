### 웹접근성과 웹표준

본격적으로 html 수업을 시작하면서 먼저 '웹(Web)'이란 무엇인지, 또 html과 xhtml의 차이점 등 이론 수업을 진행하였다.

한편으로는 들어본 적 있는 **웹접근성과 웹표준**에 대해 짚어보는 수업이였다.
사실 들어본적도 있고 지켜야 한다는건 익히 알고 있었으나 정작 실무에서나 내 개인프로젝트에서나 **"작동하기만 하면 된다"**에 치우쳐 있어서 미처 챙기지 못하는 부분이였는데 중요성을 강조하는 내용과 더불어 **"WAI-ARIA"**도 함께 알아가는 기회였다.

WAI-ARIA란?

HTML과 Xhtml의 차이점으로는
HTML의 경우 원래라면 문법 오류의 문제지만 유동성 있게 넘어가는 편에 반헤
Xhtml의 경우 규칙을 매우 엄격하게 준수해야하고 소문자만을 지원한다.

또 API의 의미를 몰랐었는데 Application Programming interface 라는 뜻을 알았고
이것은 "자바스크립트로 미리 만들어둔 기능"이라는 의미에 대해서도 알게되었다.

다시 한번 정말 의미도 모르고 그저 사용 만 해왔구나..라는 심정을 느꼈다.

```
스마트폰 view에 따른 명칭
세로보기 : 포트레이트, 가로보기 : 랜드스케이프
```

---

- html 마크업 언어에서 <br>, <hr> 등 하나로만 끝나는 마크업은 'empty element'라고 한다.

- 시작~종료 태그 모두 포함한 마크업은 element라고 칭한다. <div></div> => element

- 속성(attribute)는 마크업 안의 추가적인 기능을 부여하는 값을 의미한다. type, id, class, required 등 이 중 **required는 논리속성**으로 여러개의 값이 아닌 오직 true와 false로만 값을 정의할 수 있는 속성을 의미한다.

### html과 xhtml의 차이점

1. 대소문자 구분
   => html은 두가지 방식 모두 호환하지만 xhtml은 소문자만을 호환한다.
   => 하지만 html에서도 소문자 위주로 작성한다.

2. 빈 요소(empty element)
   => xhtml에서는 종료 시 </> 형식으로 마무리한다. 이 형식은 React 내에서 JSX 문법으로 채택되어 있어 html 작성 시에도 종료 시 </>를 사용한다.

=> 논리속성은 속성 값을 지정 또는 생략 가능하다.

=> 속성값은 인용부호를 생략하거나 따옴표로 구분한다.

=> 시작태그와 동일한 속성 중복사용이 불가하다.

=> 특수문자는 Entity code로 변환하고 Name 속성으로 사용할 것을 권장한다.

[사용예시]
| `&lt;` | `&#60;` | < | Less than (보다 작은) |
| `&gt;` | `&#62;` | > | Greater than (보다 큰) |
| `&amp;` | `&#38;` | & | Ampersand (앰퍼샌드) |
| `&quot;` | `&#34;` | " | Quotation mark (쌍따옴표) |
| `&apos;` | `&#39;` | ' | Apostrophe (작은따옴표) |
| `&nbsp;` | `&#160;` | | Non-breaking space (줄바꿈 없는 공백) |

- parsing = 해석
- DOCTYPE = 문서타입

경로에는 **상대경로와 절대경로** 2가지의 유형이 있는데 **Github는 절대경로를 인식하지 못한다.** 오로지 상대경로에서만 이용해야하며 복붙을 하더래도 위치변경이 되지 않는다.
