### 11월 19일

vscode를 본격적으로 사용하면서 CLI의 기본 문법을 공부하기 시작.
조금씩은 찍먹(?)해본 명령어들이기도 하고 그다지 긴 문장들이 아니여서 따라가기에 쉬었다.

IDE는 코드에디터를 의미한다.

CLI 명령어를 배우면서 폴더 구조를 배웠다.

폴더는 가계도 구조를 가지고 가는데 제일 첫번째 최상위 폴더를
'Root Directory'라고 명칭한다.

> 디렉토리 구조

    - Root Directory
      - Middel Directory
        - Basic Directory

`✨Unix 체제에 대해서 찾아보기`

|  CLI  |                기능                |
| :---: | :--------------------------------: |
|  cd   |             폴더 이동              |
| mkdir |           디렉토리 생성            |
|  RM   |                삭제                |
|  mv   | 폴더나 파일 위치 이동 및 이름 변경 |
|  cp   |                복사                |
|  ls   |      폴더 및 파일 리스트 조회      |
| echo  |     내용을 추가하여 파일 생성      |
|  cat  |          추가한 내용 조회          |

```
[유의사항]
- cd 명령어를 입력할 때 뒤 경로를 쓰지 않을 경우 무조건 Root Directory로 이동한다.
- 디렉토리 명에는 괄호 기호를 쓰지 말 것


1. '.'의 위치가 폴더명의 앞에 붙어 있을 경우 숨김파일이라는 의미
2. '.'의 위치가 뒤에 있을 경우 확장자 명
3. './'에서 '/'의 의미는 하위폴더
4. './'는 상대경로라는 의미
```

`항상 디렉토리의 위치를 조회 하는것을 중요하게!`

#### branch를 사용하는 이유

(!이해한 대로 작성)
branch란 나무처럼 몸통(기본 브랜치)가 있고, 가지(각자 부여한 브랜치)로 형태로 가지고 간다.

즉 브랜치를 구분하는 이유는 각각의 수정사항을 분리한 후 최종적으로 정리가 되었을 때 merge 하기 위함.

branch를 변경할 경우 Working Directory가 깨끗한 전제에서 branch를 변경, branch 이름을 바꾸기 전에 commit을 먼저 실행한 후에 git switch 명령어를 사용한다. 변경하기 전 status를 사용하여 상태를 확인한다.

`즉, commit 하지 않으면 switch를 하지 말아라`

또한 깃저장소는 꼭 내부폴더 구조에 넣어둬야 하고
상위폴더에는 생성하지 않는다. 또한 저장소 안에 다른 저장소가 있어도 안된다.
